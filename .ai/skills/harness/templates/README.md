# Harness Starter Templates

Starter templates for common harness shapes.

## Available packs

- `fullstack-app-starter.md`
  - Use for product engineering projects with frontend, backend, QA, and release concerns.
- `code-review-starter.md`
  - Use for review, audit, remediation planning, and finding synthesis workflows.
- `research-content-starter.md`
  - Use for research, drafting, packaging, and reviewer-separated content workflows.

## How to use

1. Audit the current project first.
2. Pick the nearest starter pack.
3. Keep the useful structure, but rewrite names, commands, outputs, and verification steps to match the actual project.
4. Generate repo artifacts from the pack instead of copying the pack file itself into the target project.

## Rule

- A starter template is a skeleton, not a contract.
- Prefer smaller adapted output over a larger but cargo-culted harness.
