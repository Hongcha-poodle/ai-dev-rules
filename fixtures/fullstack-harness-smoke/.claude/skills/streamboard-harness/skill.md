# Streamboard Harness

## Purpose

- Coordinate multi-role execution for Streamboard feature work.

## Trigger

- Use when the request affects both implementation and verification, or when the user asks to set up or extend the project harness.

## Workflow

1. Read `docs/harness/overview.md`
2. Confirm affected commands in `docs/harness/commands.md`
3. Route architecture questions to `architect`
4. Route UI work to `frontend-dev`
5. Route API/data work to `backend-dev`
6. Route verification to `qa-engineer`
7. Route deploy-readiness review to `release-engineer`

## Output Contract

- intermediate artifacts in `_workspace/`
- final summary grounded in `docs/harness/validation.md`
